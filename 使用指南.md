# 📸 可视化图片标注平台 - 使用指南

## 🎯 平台简介

**可视化图片标注平台**是专为转载数据标注工作设计的人工标注工具，支持同时查看muri、uri和网页（ord.page）三列数据，帮助标注人员快速比对图片一致性。

### 核心特性
- ✅ **双图对比**：同时显示muri和uri图片，便于快速比对
- ✅ **批量预加载**：提前下载所有图片，避免等待
- ✅ **标准标注**：完整的【转载】数据标注标准选项
- ✅ **批量操作**：支持批量标记和导出
- ✅ **离线缓存**：自动缓存图片，提升加载速度

## 🚀 快速开始

### 方法一：一键启动（推荐）

```bash
# 给予执行权限
```


## 📊 数据格式要求

输入CSV文件**必须包含**以下列：

| 列名 | 说明 | 是否必需 |
|------|------|----------|
| `muri` | 预览图片URL | ✅ 必需 |
| `uri` | 原图URL | ✅ 必需 |
| `ord.page` | 网页URL | ✅ 必需 |
| `ord.id` | 数据唯一ID | 可选（推荐） |
| `ord.source` | 数据来源 | 可选 |

**示例格式**：
```csv
ord.id,muri,uri,ord.page,ord.source
120638831,http://example.com/preview1.jpg,http://example.com/original1.jpg,https://news.example.com/article1,adobe
121531583,http://example.com/preview2.jpg,http://example.com/original2.jpg,https://news.example.com/article2,cnsphoto
```

## 🔧 标注标准说明

### 【是否有效】维度
| 选项 | 说明 | 适用场景 |
|------|------|----------|
| **有效数据** | 图片在网页中存在且一致 | 正常转载，图片一致 |
| **电影宣传图** | 电影海报或宣传图片 | 电影相关报道 |
| **有水印** | 图片带有水印 | 需要记录水印类型 |
| **无效数据** | 图片不存在或不一致 | 网页无图或图片不同 |
| **不一样的图** | 图片主题不同 | 网页图片与muri/uri不一致 |
| **空白** | 无图片或空白图片 | URL无效或图片为空 |

### 【有水印】维度
| 水印类型 | 说明 |
|----------|------|
| **无** | 无水印 |
| **视觉中国** | 视觉中国水印 |
| **新华网** | 新华网水印 |
| **中新社** | 中国新闻社水印 |
| **图虫** | 图虫水印 |
| **人民日报** | 人民日报水印 |
| **央视总台** | 央视总台水印 |
| **角标水印** | 角标形式的水印 |
| **其他水印** | 其他未列出的水印 |

## 📋 工作流程

### 步骤1：准备数据
1. 确保CSV文件格式正确
2. 检查图片URL是否有效
3. 确认文件编码为UTF-8

### 步骤2：启动平台
```bash
./一键启动标注平台.sh
```

### 步骤3：加载数据
1. 点击 **"加载CSV数据"** 按钮
2. 选择你的数据文件
3. 系统会自动验证数据格式

### 步骤4：批量预加载（可选但推荐）
1. 点击 **"批量预加载图片"** 按钮
2. 系统会并发下载所有图片到本地缓存
3. 等待下载完成（进度显示在状态栏）
4. 完成后所有图片会快速加载显示

### 步骤5：开始标注
1. 使用导航按钮浏览图片
2. 对比muri和uri图片的一致性
3. 根据标准选择标注选项：
   - 选择 **【是否有效】** 类别
   - 如有水印，选择 **【有水印】** 类型
   - 在备注框中添加额外信息
4. 标注会自动保存到内存

### 步骤6：批量操作
- **批量标记**：快速标记大量相似数据
- **ID跳转**：直接定位到特定记录
- **随机查看**：随机抽样检查

### 步骤7：保存和导出
1. 点击 **"保存标注"** 保存为JSON格式
2. 点击 **"导出结果"** 导出为CSV格式
3. 结果文件包含原始数据和标注信息

## 🔍 高级功能

### 1. 批量预加载
```bash
# 平台会自动执行
- 多线程并发下载
- 失败自动重试
- 本地缓存优化
- 断点续传支持
```

### 2. 离线缓存
- 所有下载的图片会自动缓存到 `~/.image_annotation_cache`
- 下次打开相同URL时直接从缓存加载
- 可手动清理缓存目录

### 3. 批量标注
- 支持批量标记所有记录为相同类别
- 支持清除所有标注重新开始
- 支持按ID精确跳转

### 4. 结果导出
**JSON格式**：完整标注数据，包含元信息
**CSV格式**：原始数据 + 标注列，便于导入Excel

## 🛠️ 故障排除

### 常见问题1：图片加载失败
```
原因：网络问题或URL无效
解决：
1. 检查网络连接
2. 确认URL是否有效（浏览器中打开测试）
3. 使用批量预加载功能（支持重试）
4. 检查代理设置
```

### 常见问题2：平台启动失败
```
原因：缺少依赖
解决：
1. 运行：pip install pillow opencv-python requests pandas numpy
2. 确保有tkinter：sudo apt-get install python3-tk
3. 检查Python版本：python3 --version
```

### 常见问题3：数据加载错误
```
原因：CSV格式问题
解决：
1. 确保文件为UTF-8编码
2. 检查必需的列名是否正确
3. 检查URL格式是否完整
4. 使用示例数据测试
```

### 常见问题4：内存不足
```
原因：图片太多或太大
解决：
1. 减少批量预加载的线程数
2. 分批处理大数据文件
3. 清理缓存目录
4. 增加系统内存
```
